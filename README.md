# 心理热线值班表系统

专业、高效的心理热线排班管理单页应用。

## 功能特性

- 🔐 **角色权限系统**：咨询师、热线主管、排班员三种角色，不同身份看到不同的功能入口
- 📅 **可视化值班表**：按周展示值班安排，支持多个热线和班次
- ✋ **拖拽排班**：拖拽咨询师卡片到班次进行排班
- 🛡️ **资质校验**：自动验证咨询师是否具备对应专线的资质
- 🌙 **夜班双人值守**：夜班强制要求至少2人值守
- 📋 **班次详情面板**：点击班次查看详细信息、已排班人员、资质要求

## 技术栈

- React 18 + TypeScript
- Vite 构建工具
- React DnD 拖拽库
- 纯 CSS 样式（无框架依赖）

## 快速启动

### 方式一：本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

访问 http://localhost:3000

### 方式二：Docker 容器启动

```bash
# 构建并启动容器
docker-compose up -d

# 查看日志
docker-compose logs -f web

# 停止容器
docker-compose down
```

访问 http://localhost:3000

## 核心业务规则

### 1. 资质校验规则
- 咨询师必须具备对应热线所需的资质才能被排到该专线
- 资质不匹配时显示错误提示，阻止排班
- 已排班人员列表中会用红色标记资质不匹配的人员

### 2. 夜班双人值守规则
- 夜班（00:00-08:00）必须至少安排 2 人
- 不足 2 人时在班次详情面板显示警告
- 值班表中班次槽位会显示「还需 X 人」提示

### 3. 冲突检测规则
- 同一咨询师不能在同一时段被排到多个热线
- 检测到冲突时显示错误提示

## 排错线索

### 🔴 常见问题

**Q: 页面空白，控制台报错 "React is not defined"**
- 确保已安装依赖：`npm install`
- 检查 Node 版本 >= 16

**Q: 拖拽功能不工作**
- 确认已使用排班员或主管身份登录
- 检查浏览器控制台是否有 React DnD 相关错误
- 确认 `react-dnd` 和 `react-dnd-html5-backend` 已正确安装

**Q: 资质校验不生效**
- 检查 `src/utils/validation.ts 中的校验逻辑
- 确认咨询师的 qualifications 数组包含正确的资质 ID
- 确认热线的 requiredQualification 配置正确

**Q: 夜班双人值守不提示**
- 检查班次的 isNightShift 属性是否为 true
- 检查 requiredCounselors 是否设置为 2
- 点击班次打开详情面板查看警告

**Q: Docker 启动失败**
- 检查 3000 端口是否被占用：`lsof -i :3000`
- 重新构建镜像：`docker-compose build --no-cache`
- 查看容器日志：`docker-compose logs web`

**Q: Smoke 测试失败**
- 确保应用已在 http://localhost:3000 启动
- 检查 puppeteer 是否正确安装
- 尝试手动执行测试步骤验证功能

### 🟡 调试技巧

1. **查看应用状态：打开浏览器开发者工具 → Application → Local Storage
2. **查看网络请求：Network 面板
3. **查看组件状态：安装 React DevTools 扩展
4. **查看拖拽事件：在控制台输入 `window.__REACT_DEVTOOLS_GLOBAL_HOOK__`
