# 心理热线值班表系统

专业、高效的心理热线排班管理单页应用。

## 功能特性

- 🔐 **角色权限系统**：咨询师、热线主管、排班员三种角色，不同身份看到不同的功能入口
- 📅 **可视化值班表**：按周展示值班安排，支持多个热线和班次
- ✋ **拖拽排班**：拖拽咨询师卡片到班次进行排班
- 🛡️ **资质校验**：自动验证咨询师是否具备对应专线的资质
- 🌙 **夜班双人值守**：夜班强制要求至少2人值守
- 📋 **班次详情面板**：点击班次查看详细信息、已排班人员、资质要求

## 技术栈

- React 18 + TypeScript
- Vite 构建工具
- React DnD 拖拽库
- 纯 CSS 样式（无框架依赖）

### 失败用例：资质不匹配时拖拽排班失败

**场景描述**：将不具备「危机干预专线」资质的咨询师拖拽到「危机干预专线」班次时，排班失败并给出详细错误原因。

**测试步骤**：
1. 启动应用：`npm run dev`
2. 打开 http://localhost:3000
3. 选择「王排班员」身份登录（排班编辑权限）
4. 点击左侧菜单「排班编辑」
5. 从左侧咨询师列表找到 **张小明**（仅具备「普通心理热线」资质，无危机干预资质）
6. 将张小明拖拽到任意「危机干预专线」（红色标签）的班次槽位

**预期结果（失败）**：
- ❌ 张小明 **不会** 被添加到该班次
- ❌ 页面顶部弹出红色错误提示：`张小明 不具备 危机干预专线 所需的资质 - 需要：危机干预专线资质，当前具备：普通心理热线资质`
- ❌ 该班次人员列表保持不变

**验证要点**：
- 错误提示必须包含**完整原因**：既说明不具备资质，又列出"需要的资质"和"当前具备的资质"
- 人员不会被错误地添加到班次中
- 控制台无 JavaScript 报错

## 核心业务规则

### 1. 资质校验规则
- 咨询师必须具备对应热线所需的资质才能被排到该专线
- 资质不匹配时显示错误提示，阻止排班
- 已排班人员列表中会用红色标记资质不匹配的人员

### 2. 夜班双人值守规则
- 夜班（00:00-08:00）必须至少安排 2 人
- 不足 2 人时在班次详情面板显示警告
- 值班表中班次槽位会显示「还需 X 人」提示

### 3. 冲突检测规则
- 同一咨询师不能在同一时段被排到多个热线
- 检测到冲突时显示错误提示

## 排错线索

### 🔴 常见问题

**Q: 页面空白，控制台报错 "React is not defined"**
- 确保已安装依赖：`npm install`
- 检查 Node 版本 >= 16

**Q: 拖拽功能不工作**
- 确认已使用排班员或主管身份登录
- 检查浏览器控制台是否有 React DnD 相关错误
- 确认 `react-dnd` 和 `react-dnd-html5-backend` 已正确安装

**Q: 资质校验不生效**
- 检查 `src/utils/validation.ts 中的校验逻辑
- 确认咨询师的 qualifications 数组包含正确的资质 ID
- 确认热线的 requiredQualification 配置正确

**Q: 夜班双人值守不提示**
- 检查班次的 isNightShift 属性是否为 true
- 检查 requiredCounselors 是否设置为 2
- 点击班次打开详情面板查看警告

**Q: Docker 启动失败**
- 检查 3000 端口是否被占用：`lsof -i :3000`
- 重新构建镜像：`docker-compose build --no-cache`
- 查看容器日志：`docker-compose logs web`

**Q: Smoke 测试失败**
- 确保应用已在 http://localhost:3000 启动
- 检查 puppeteer 是否正确安装
- 尝试手动执行测试步骤验证功能

### 🟡 调试技巧

1. **查看应用状态：打开浏览器开发者工具 → Application → Local Storage
2. **查看网络请求：Network 面板
3. **查看组件状态：安装 React DevTools 扩展
4. **查看拖拽事件：在控制台输入 `window.__REACT_DEVTOOLS_GLOBAL_HOOK__`
